catch-exception library has new home

08 listopad 2014

Rod Woo, original author of catch-exception library decided to not support his project. This is related to Java 8’s lambda expressions, which mostly simplified exception handling.

Because many projects and companies are still using Java 7, we want to fix some problems, like compatibility with AssertJ 1.7.0 and Mockito 1.10.0. We only want to maintain catch-exception and fix existing problems. Supported version was reborn at new home https://github.com/Codearte/catch-exception! We hope that the project will be equally as good as the original version.

For creating new projects built on Java 8 please do not use this library! There is good article about testing exception with Java8: jUnit: Testing exception with java 8 and lambda expressions by Rafa≈Ç Bobrowiec

Changes
—

  • New project home github.com/Codearte/catch-exception

  • AssertJ was replaced by FEST Fluent Assertions.
    FEST Fluent Assertions is dead project.

  • New class com.googlecode.catchexception.apis.BDDCatchException was introduced for BDD style exception handling. Naming convention is consistent with BDDAssertions and BDDMockito.

  • Class com.googlecode.catchexception.apis.CatchExceptionAssertJ was declared deprecated,
    use com.googlecode.catchexception.apis.BDDCatchException instead.

  • Class com.googlecode.catchexception.apis.CatchExceptionBdd was deprecated and replaced with BDDCatchException. Please switch FEST Fluent Assertion with AssertJ.

  • Added support for AssertJ 1.7.0

  • Added support for Mockito versions from 1.8.1 to 1.10.8

  • Discussion group and old issues page was replaced by new Github Issues Page

  • Maven coordinates was changed from com.googlecode.catch-exception:catch-exception to
    eu.codearte.catch-exception:catch-exception
